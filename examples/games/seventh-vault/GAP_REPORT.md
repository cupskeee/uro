# GAP REPORT — The Seventh Vault

## 1. Summary

Uro's current surface supported the heist's **spine** startlingly well — one shared campaign
with four WS clients receiving byte-identical narration, a declarative alarm that escalates off
committed events, a Chronicler trust gate that did exactly what D-32 promises (the tier-0 guard
died as canon, the tier-3 Warden's "death" became `truth=unknown` testimony, the rumor decayed
0.9→0.495 across a `knows` hop, and the zero-witness scuffle propagated silence), and the whole
20-beat multiplayer arc replays **byte-identically** with no key. The single biggest wall:
**the deterministic mode cannot reach Uro's mechanics at all** — the planner is an LLM stage
and the stub planner is a constant no-op, so no check ever resolves, no encounter can start,
and (compounding it) a check that IS forced through commits **no event**, `BeatResult` exposes
only a check *count*, and the beat RNG keys off the fresh-per-run `campaign_id` rather than the
stored seed. Every consequence the heist turns on therefore had to be driven by rules
pattern-matching exact intent strings, host-authored events, or the game's own dice. Runner-up:
**there is no management surface** — 10/10 lobby/state calls went through the embedded library,
so a non-Python client could boot nothing (S4). Play also flushed out one genuine engine bug:
`PartyArbiter` misrotates the token backward when the turn-holder disconnects (G-17).
Non-issues worth naming: the D-32 protection ceiling and participant-scope rules never blocked
the game (designing *around* them produced the better story), outcome bundles were cleanly
idempotent, the turn token is race-safe in-process (S3), and the one-ruleset-per-process limit
never actually bit (S6 — moot while mechanics are unreachable anyway). Two disclosures for the
record: TASK Section 3's fourth score state `caught` ships as `betrayed` — the two endings the
TASK's own Stage 7/DoD mandate are clean-escape and double-cross, nobody is ever captured, and
no committed state records guard-capture (the abandoned crew's fate exists as the committed
legend rumor, not a status); and the `out/` evidence artifacts cited below are REGENERATED by
`./run.sh --stress`, not committed (the sibling games' convention) — the gap table's Evidence
column cites committed source `file:line` throughout.

## 3. TOP 3 THINGS URO MUST ADD for this game to be good

1. **A REST management surface** — tied to gap rows **G-15** (with G-14, G-16, G-19). Without
   it this "network multiplayer game" is a Python library program wearing a WebSocket hat: the
   host process must embed the engine for world creation, seating, discovery, every HUD read,
   and the epilogue clock, and the campaign id travels by FILE. Ten enumerated endpoints, each
   with a captured 404, are in `out/stress/s4_management.txt`. This is the difference between
   "Uro has a server" and "Uro is a server."
2. **A deterministic path into the mechanics gate, with committed outcomes** — tied to
   **G-1** (with G-2, G-3, G-7). The engine's own signature claim ("the ruleset rolls from a
   seeded RNG through the beat pipeline") is unreachable in the only mode CI can run: the
   planner needs a live LLM, a forced check commits no event and is invisible in `BeatResult`
   and over the wire, the RNG can't be pinned across runs, and no affordance can move an item.
   Until then, every deterministic consumer must build its consequences out of intent-string
   pattern-matching and host-authored events — which is exactly what this game did, and it
   shouldn't have had to.
3. **Arbiter shapes beyond round-robin (and fix the departure bug)** — tied to
   **G-9/G-10/G-11/G-12/G-8**, plus **G-17/G-18**. The heist wanted five specific shapes —
   simultaneous/parallel, proposal-window (QUEUED is already reserved), consensus/vote,
   reactive/interrupt, and consensual-PvP — and every one reduced to "burn a strict-rotation
   turn or don't do it." The OQ-7 port is ready for them; meanwhile the one arbiter that exists
   misrotates when a holder disconnects and forgets the table order on reconnect.

## 4. Verdicts on the targeted leftover-work

- **S1 Arbiter beyond round-robin (OQ-7):** **Hit** (four probes over the live WS channel +
  the arc's betrayal). The game needs, concretely: a **simultaneous/parallel** arbiter (G-9),
  a **proposal-window** arbiter (G-10 — `AdmitDecision.QUEUED` is reserved for exactly this
  and unimplemented), a **consensus/vote** arbiter (G-11), a **reactive/interrupt** arbiter
  (G-12), and a **consensual-PvP** protocol (G-8). Verdict: the deferral was right for the
  PoC, but this game is now a real consumer for at least QUEUED + interrupt; round-robin
  forced the entire script into strict rotation and cost the lookout, the vote, and a played
  (rather than authored) betrayal.
- **S2 Party co-combat + PC-anchored recall:** **Hit — PARTIAL.** Attribution is real: every
  beat runs as the acting participant's own PC (P7 threading held across all 20 beats and both
  probes). Vantage is **absent by construction**: `assemble_recall` has no PC parameter, so a
  split crew shares one omniscient memory (G-13). Party co-combat is unreachable (G-26 — no
  encounter under stub; single-aggressor auto-resolve live), and the Chronicler-external
  skirmish is the working answer — arguably the intended one. Verdict: recall anchoring is now
  blocking a real multiplayer fantasy; co-combat can stay external if that posture is
  documented as the answer.
- **S3 Party-race / turn-token edge:** **Hit — race-safe** (8 contested rounds, both clients
  firing simultaneously: exactly one commit per round, losers got clean `not_your_turn`,
  nothing double-committed, nothing lost). The atomicity is cooperative-scheduling luck (no
  lock, no await between read and decide) — fine for one process, and the P7 `expected_head`
  deferral becomes real the day the server scales past it. Verdict: deferral right.
- **S4 Missing REST management surface:** **Hit — the enumerated list** (all captured as 404s
  in `out/stress/s4_management.txt`): `POST /worlds`, `POST /worlds/{w}/campaigns`,
  `POST /campaigns/{c}/join`, `GET /campaigns`, `GET /campaigns/{c}/roster`,
  `GET /campaigns/{c}/state`, `GET /campaigns/{c}/items/{i}`, `GET /campaigns/{c}/sheets/{a}`,
  `GET /campaigns/{c}/chronicle`, `POST /campaigns/{c}/time-skip` — each row lists the
  `store.*` call used instead. A network-only client **could not have run this game at all**:
  it can play beats and post outcomes, nothing else. Verdict: the deferral is now the single
  biggest blocker for any non-Python consumer (Top-3 #1).
- **S5 Session lifecycle:** **Hit — and it found an engine bug.** What broke: (1) a
  turn-holder's mid-heist disconnect misrotates the token BACKWARD (G-17, `session.py:124`
  off-by-one — p1 got a double turn, p3 was skipped); (2) a reconnect on the same token reseats
  the player at the END of the ring — table order mutated by transport noise (G-18); (3) a late
  5th player can be seated in committed state but never authenticated (G-19). What survived:
  the PC binding (event-sourced), exactly as designed. Verdict: "turn state is session-only"
  was a reasonable PoC line, but it now loses real state (order) and has a real bug; fix
  `note_left`, then event-source the ring.
- **S6 One-ruleset-per-server-process:** **Hit.** The limit is real and cleanly enforced (the
  uro-pbta campaign on the uro-basic server got the documented `beat_failed`, G-20). But it did
  **not** constrain this game: under the stub provider no ruleset's mechanics are reachable
  (G-1), so the choice of ruleset was cosmetic. Verdict: deferral right; fix G-1 first, or a
  second ruleset buys nothing.
- **S7 Reaction Layer has no counters (the alarm):** **Hit — the wall is triple** (all three
  captured in `stress/s7_counters.py`): the rule we wanted —
  `trigger CheckResolved{outcome:failure} → increment heat; heat>=3 → lockdown` — fails
  because (1) no check-outcome event exists (and the trigger VALIDATES silently — the
  accepted-but-inert footgun), (2) no counters/arithmetic exist (G-4), (3) the alarm can't even
  be named `suspicious` (closed ThreadState literal, G-5). The enum-ladder workaround is doubly
  lossy: distance-to-lockdown is inexpressible, and each rung binds to an exact intent string a
  one-word script edit silently disarms. The scope-splitting wrinkle also bit (G-6: alarm rule
  + "spotted" edge = two scopes, and the edge fails even split). Verdict: for THIS game the
  ladder sufficed only because the players are scripted; for free-text players the alarm is
  unbuildable declaratively — this is the sharpest WASM-tier/counter evidence the refusal log
  (rule_pack.py, printed each run) can offer.
- **S8 No game↔world time mapping:** **Hit.** The invented mapping: *one heist = one night =
  zero world-days*, and *the epilogue = `agenda_tick(branch, 7)` called by hand* in arc.py.
  Sharper than expected: beats never advance world time under ANY provider — `BeatPlan.
  time_cost` is a dead field the planner is prompted to emit and nothing consumes (G-24), so
  the zero-duration night is an engine property, not a stub artifact. Also measured:
  `duration_rounds` in the outcome bundle has no time semantics, and a 14-day tick fires a
  7-day agenda ONCE (once-per-skip, not per-boundary — G-24). Verdict: livable for a one-shot;
  any campaign where time emerges from play would shadow the whole calendar in game code.
  Deferral acceptable, semantics worth documenting.

## 5. Appendix — the refusal log (S7 evidence, printed live by every `arc.py --print-log` run)

Six rules the heist wanted verbatim and the grammar could not express — the wished-for
`rule_pack` JSON, the missing primitive, and where game code carries the logic instead — are
registered in [`rule_pack.py`](rule_pack.py) `register_refusals()`: **RL-1** the heat meter
(counter+threshold), **RL-2** escalate-on-any-failed-check (no check event), **RL-3** the
heist's own state vocabulary (closed ThreadState literal), **RL-4** alarm+spotted in one rule
(single-dimension scope; the split *still* fails add_edge's both-endpoints rule), **RL-5** a
lockdown that lifts after quiet days (no relative time), **RL-6** a weighted guard-response
table (no randomness — deliberately; the game's own dice + Chronicler is the sanctioned
answer, and it worked).
